## How to use these

These are templates, not scripts. Substitute your own system details.

The most useful practice: read the example once, close it, then answer the same question about your own project from scratch. The structure should be internalized, not memorized.

Each example includes annotated callouts explaining *why* the answer is structured the way it is — what signal it's sending to the interviewer.
