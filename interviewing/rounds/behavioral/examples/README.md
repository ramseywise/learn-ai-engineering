# Behavioral Round — Worked STAR Examples

Each file is a complete STAR+Reflection story with annotations on what makes each
component strong. Practice by reading the scenario first, drafting your own version of
that story category, then comparing structure (not content) against the example.

| Topic | File | Story category |
|-------|------|---------------|
| Technical conflict resolution | [conflict-resolution.md](conflict-resolution.md) | Conflict with a peer — technical disagreement, steelman, resolution via data |
| Real failure with behavior change | [failure-owned.md](failure-owned.md) | Failure/mistake owned — real stakes, causal role stated, visible behavior change |
| Delivering with incomplete requirements | [ambiguity-rescued.md](ambiguity-rescued.md) | Ambiguous scope — assumption documentation, reversible build, iteration |

## How to use these examples

1. **Don't borrow the story** — these are structural templates, not content to lift. Your interviewer may have read the same resources.
2. **Study the annotations** — each component has a note on what signal it sends and what the common mistake is at that point.
3. **Map your own story** — after reading an example, write out your equivalent story in the same structure. Does your Action section dominate? Does your Result have a number? Does your Reflection name a specific behavior?
4. **Check the anti-patterns** — [study-guide.md](../study-guide.md) §Anti-patterns maps to things that can go wrong at each STAR component.
