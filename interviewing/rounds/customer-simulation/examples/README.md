# Customer Simulation — Worked Examples

Three fully annotated scenario dialogues. Each shows the conversation flow with inline notes on what makes each response effective (or what the failure mode would be).

Read these after internalizing the six habits in `study-guide.md` — the annotations reference the habits by name. The goal is to see the habits in motion, not to memorize the scripts. The interviewer will improvise; these are demonstrations of the underlying structure.

## Files

- **`incident-response.md`** — "Your AI gave wrong information to a customer."
  Tests: crisis empathy, ownership, containment instinct, trust repair.

- **`scope-negotiation.md`** — "We want everything your competitor's demo showed."
  Tests: scope discipline, discovery under pressure, boundary-setting without "no."

- **`bad-news-delivery.md`** — "The pilot metrics look bad and renewal is next month."
  Tests: bad-news honesty under commercial pressure, recovery framing, commitment calibration.

## How to use these

1. Read the dialogue once straight through.
2. Cover the annotations and re-read — try to identify what each response is doing before looking.
3. Run the scenario out loud yourself. Note where your instinct diverges from the example.
4. The divergence is the practice target — not a failure, but a signal.
